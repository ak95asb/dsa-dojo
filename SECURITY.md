## Supported Versions

We release patches for security vulnerabilities for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in DSA Dojo, please report it by:

1. **Opening a security advisory** on GitHub: [Security Advisories](https://github.com/ak95asb/dsa-dojo/security/advisories)
2. **Or sending an email** to the maintainer (check GitHub profile for contact)

Please include:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if you have one)

### What to Expect

- **Acknowledgment**: We'll acknowledge your report within 48 hours
- **Updates**: We'll keep you informed about our progress
- **Credit**: We'll publicly credit you for the discovery (unless you prefer to remain anonymous)
- **Fix**: We'll work to release a fix as quickly as possible

## Security Best Practices

When using DSA Dojo:
- Keep your installation up to date
- Be cautious when importing custom problem libraries from untrusted sources
- Review generated code before execution
- Don't share your database file publicly (it may contain personal progress data)

## Disclosure Policy

- Security issues will be disclosed responsibly
- We'll coordinate disclosure timing with reporters
- Patches will be released before public disclosure
